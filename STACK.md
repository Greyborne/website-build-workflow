# Stack: Next.js 15 + Payload CMS v3 + Tailwind CSS

## Why This Stack

### The Problem With WordPress
WordPress gives you a content admin panel fast, but chains you to:
- Plugin dependency hell (security, updates, conflicts)
- PHP template system that fights you on custom layouts
- Performance overhead that requires caching plugins to fix
- Themes that impose their design opinions on your work
- Hosting requirements that make you overpay for what you need

### The Solution
| Layer | Tool | Why |
|---|---|---|
| Framework | Next.js 15 (App Router) | Production-grade React, SSR/SSG, image optimization, built-in routing |
| CMS | Payload CMS v3 | Admin panel built into Next.js; TypeScript-native; no separate service needed |
| Styling | Tailwind CSS v4 | Utility-first, design tokens as CSS variables, no fighting cascade |
| Language | TypeScript | Catches bugs before runtime, required by Payload v3 |
| Hosting | Hostinger VPS | Node.js runtime, full control, affordable |
| Database | PostgreSQL (via Hostinger or self-hosted) | Payload v3 default; also supports SQLite for dev |

### What You Get
- Content editors see a clean admin panel at `yourdomain.com/admin`
- They can create/edit pages, posts, team members, services, etc.
- You define the schema in code — they can't break the layout
- Frontend is completely custom React/Next.js — no theme limitations
- One repo, one deployment, one process

## Quickstart (New Project)

### Prerequisites
- Node.js 20+
- PostgreSQL database (local or remote)

### 1. Create the Project

```bash
npx create-payload-app@latest my-site
# Choose: Next.js template
# Choose: PostgreSQL
```

### 2. Configure Environment

```bash
cp .env.example .env.local
```

Edit `.env.local`:
```env
DATABASE_URI=postgresql://user:password@localhost:5432/mysite
PAYLOAD_SECRET=your-random-secret-minimum-32-chars
NEXT_PUBLIC_SERVER_URL=http://localhost:3000
```

### 3. Install Tailwind CSS v4

```bash
npm install tailwindcss @tailwindcss/postcss postcss
```

Create `postcss.config.mjs`:
```js
export default {
  plugins: {
    '@tailwindcss/postcss': {},
  },
}
```

In `src/styles/globals.css`:
```css
@import "tailwindcss";

/* Design tokens go here — generated by p05 prompt */
:root {
  --color-primary: oklch(55% 0.2 240);
  /* ... */
}
```

### 4. Start Development

```bash
npm run dev
# App: http://localhost:3000
# Admin: http://localhost:3000/admin
```

## Hostinger VPS Deployment

### Server Requirements
- Hostinger KVM 2 or higher (2 CPU, 8GB RAM recommended)
- Ubuntu 22.04 LTS
- Node.js 20+ (via nvm)
- PostgreSQL 15+
- Nginx (reverse proxy)
- PM2 (process manager)

### Deployment Steps

#### 1. Install Runtime
```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
nvm install 20
npm install -g pm2
```

#### 2. Install PostgreSQL
```bash
sudo apt install postgresql postgresql-contrib
sudo -u postgres createdb mysite
sudo -u postgres createuser mysiteuser --pwprompt
sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE mysite TO mysiteuser;"
```

#### 3. Clone and Build
```bash
git clone https://github.com/Greyborne/YOUR-SITE.git /var/www/mysite
cd /var/www/mysite
npm install
cp .env.example .env.local
# Edit .env.local with production values
npm run build
```

#### 4. Start with PM2
```bash
pm2 start npm --name "mysite" -- start
pm2 startup
pm2 save
```

#### 5. Nginx Config
```nginx
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

```bash
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
```

## Multi-Site MSP Setup

For offering this as a service, run each client site as a separate app on the same VPS:

```
/var/www/
├── client-a/       ← port 3001
├── client-b/       ← port 3002
└── client-c/       ← port 3003
```

Each with its own Nginx server block and PostgreSQL database. Use PM2 to manage all processes:

```bash
pm2 list
# client-a    online
# client-b    online
# client-c    online
```

```ts
import { CollectionConfig } from 'payload'

export const Services: CollectionConfig = {

    { name: 'featured', type: 'checkbox', defaultValue: false },
  ],
}
```

Editors see a clean form. You control the frontend display completely.

## When to Use SQLite Instead

For small sites, prototypes, or local dev, SQLite is simpler:

```env
DATABASE_URI=file:./payload.db
```

SQLite works fine for sites with < 10 editors and < 50k records.
Switch to PostgreSQL before going to production for client sites.
