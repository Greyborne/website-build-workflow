# Examples

This folder contains completed workflow walkthroughs that demonstrate each phase of the process end-to-end.

Each example shows the actual prompts used, the AI output received, and the decisions made during review.

---

## How to Use Examples

1. Read the `brief.md` to understand the project context
2. Read the `inspiration/` analyses to see what patterns were extracted
3. Read `design-system.md` to see how tokens were derived from inspiration
4. Read `component-inventory.md` to see how components were defined
5. Read `page-blueprints.md` to see how sections were planned

Use these as reference when your AI output seems too generic — compare the specificity level of your output to the example output.

## Adding Your Own Examples

After completing a project, consider adding a sanitized version here (remove client-specific contact info, replace real URLs with example.com):

```bash
mkdir -p examples/{project-slug}
cp projects/{project-slug}/brief.md examples/{project-slug}/brief.md
# etc.
```

Good examples make the workflow better for future projects.
