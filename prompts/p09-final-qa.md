# P09 — Final QA Review
## Phase 7: Full-site quality check before launch

---

> **When to use:** When all pages and components are built, all copy is written, and the site runs locally without errors.
>
> **How to use:** Paste the prompt below into your AI tool. Provide the site URL (localhost or staging) for tools with browsing, or paste in code/screenshots for non-browsing models.
>
> **Also use:** Run the manual checklist at the bottom of this file yourself — some checks require actual browser interaction that AI can't reliably perform.

---

## Prompt (copy from here)

---

Perform a final pre-launch quality audit on the following website.

**Site URL (if you have browsing):** `{http://localhost:3000 or staging URL}`

**Or paste key page code/screenshots below for non-browsing review:**
```
{Content if no browsing available}
```

**Project brief summary:**
```
Client: {NAME}
Stack: Next.js 15 + Payload CMS v3 + Tailwind CSS v4
Pages built: {list all pages}
Target audience: {AUDIENCE}
Primary CTA: {PRIMARY CTA ACROSS SITE}
```

Review all of the following categories. Report every finding with:
- Category
- Severity (🔴 BLOCKING / 🟡 IMPORTANT / 🟢 POLISH)
- Location (page + section)
- Specific fix

---

### CATEGORY 1: DESIGN CONSISTENCY

- Is the design system applied consistently across all pages and components?
- Do any pages feel visually inconsistent with others?
- Is spacing consistent (always from the scale, never arbitrary)?
- Are there any components that use colors not from the defined palette?
- Does every page use the same navigation and footer components?
- Is the typographic hierarchy consistent across all pages?
- Are interactive states (hover, focus, active) consistent across all interactive elements?

### CATEGORY 2: ANTI-SLOP FINAL CHECK

Walk through the entire site as if seeing it for the first time. Answer:
- Could this website's design be mistaken for any other website in this industry?
- Are there any sections that look like they were pulled from a template?
- Does any copy read like it was generated by AI?
- Is every section on every page justified by the page blueprint — or are there sections that exist "just because"?
- Is the visual identity distinctive enough to be recognizable out of context?

### CATEGORY 3: RESPONSIVE DESIGN

- Does every page render correctly at 375px (iPhone SE), 768px (tablet), 1280px (desktop), 1440px (wide desktop)?
- Are there any horizontal scrollbars at any breakpoint?
- Is touch target size sufficient on mobile? (minimum 44×44px for interactive elements)
- Does navigation work correctly on mobile?
- Do images scale correctly without distortion?
- Is text readable (minimum 16px body, sufficient contrast) at all breakpoints?

### CATEGORY 4: ACCESSIBILITY

- Does the page have correct heading hierarchy (one H1 per page, logical H2→H3→H4 nesting)?
- Do all images have meaningful alt text (not "image", not filename)?
- Do all interactive elements have visible focus states?
- Are all form fields labeled (with `<label>` or `aria-label`)?
- Are icon-only buttons labeled (with `aria-label`)?
- Is the color contrast ratio sufficient? (4.5:1 for normal text, 3:1 for large text)
- Does the site work with keyboard-only navigation?
- Are there any `tabindex` values greater than 0 (anti-pattern)?
- Is `prefers-reduced-motion` respected for all animations?

### CATEGORY 5: PERFORMANCE

- Do all pages pass Core Web Vitals thresholds? (LCP < 2.5s, CLS < 0.1, FID/INP < 200ms)
- Are images using Next.js `<Image>` component with width/height specified?
- Are fonts loading efficiently (preconnect + display=swap)?
- Is there any layout shift caused by late-loading fonts or images?
- Are third-party scripts (analytics, chat widgets) deferred or loaded async?
- Is the build output size reasonable? (Check with `npm run build`)

### CATEGORY 6: SEO

- Does every page have a unique, descriptive `<title>` tag?
- Does every page have a unique meta description (150–160 characters)?
- Is there an OG image for social sharing on every page?
- Is there a valid `sitemap.xml` at `/sitemap.xml`?
- Is there a `robots.txt`?
- Are canonical URLs set correctly?
- Is structured data (JSON-LD) present on appropriate pages?
- Is the site indexable? (Check: `<meta name="robots" content="noindex">` should NOT be present on public pages)

### CATEGORY 7: FUNCTIONALITY

- Do all forms submit successfully and show appropriate success/error states?
- Do all navigation links go to the correct destinations?
- Are there any broken links or 404s?
- Does the Payload CMS admin panel work at `/admin`?
- Can a content editor create, edit, and delete content without issues?
- Are all CMS fields visible and functional in the admin panel?
- Does content edited in the CMS reflect on the frontend (check revalidation)?
- Are there any console errors on any page?

### CATEGORY 8: COPY FINAL REVIEW

- Is every page free of copy anti-patterns (from P08)?
- Is all placeholder/lorem ipsum copy replaced with real content?
- Is the tone consistent across all pages?
- Are CTAs specific and outcome-oriented on every page?
- Is contact information accurate (phone, email, address)?
- Is the copyright year correct in the footer?
- Are there any typos or grammatical errors?

### CATEGORY 9: SECURITY & CONFIGURATION

- Are environment variables not exposed to the client (`NEXT_PUBLIC_` prefix only where intentional)?
- Is the Payload secret key a strong, random string in production?
- Is the admin panel protected and not indexable?
- Are file upload types restricted in Payload media collections?
- Is the site using HTTPS in production?

## Manual QA Checklist (Do These Yourself in a Real Browser)

These require hands-on testing that AI cannot reliably perform:

- [ ] Tab through the entire site with keyboard only — confirm nothing is skipped or trapped
- [ ] Test contact form with real email address — confirm receipt
- [ ] Test the site with a screen reader (macOS VoiceOver: Cmd+F5, Windows: NVDA free)
- [ ] Load the site on a real mobile device (not just browser DevTools)
- [ ] Test on Chrome, Firefox, and Safari
- [ ] Run Lighthouse audit in Chrome DevTools for each page
- [ ] Check Google Search Console after indexing for any issues
- [ ] Test Payload admin on mobile browser
- [ ] Verify SSL certificate is valid and auto-renewing (Let's Encrypt via Certbot)
- [ ] Confirm PM2 restarts the app after a server reboot (`pm2 startup && pm2 save`)
- [ ] Test the site with browser zoom at 200% (accessibility requirement)
- [ ] Verify all external links open in a new tab (`target="_blank"` + `rel="noopener noreferrer"`)
